## 1. 개요

### 1.1 배경
뮤지컬 팬덤 사이에서는 회차별 캐스팅(더블/트리플 캐스팅)이 다른 공연을 여러 번 관람하며, "누구를 몇 번 봤는지", "어느 회차에 어떤 좌석에서 봤는지"를 개인적으로 기록·관리하는 문화가 있다. 레퍼런스 사이트(사의찬미 트래커)는 이런 니즈를 정적 웹페이지 + 로컬 데이터 저장 방식으로 충족시키는 툴이다.

본 프로젝트는 동일한 컨셉을 뮤지컬 〈사칠〉(2026년 재연, 링크아트센터 드림 2관)에 맞춰 제작한다.

### 1.2 목적
사용자(관객)가
1. 전체 공연 회차와 회차별 캐스팅을 한눈에 확인하고,
2. 자신이 예매/관람한 회차를 체크·기록하며,
3. 배우별로 자신이 몇 회씩 관람했는지 자동 집계하고,
4. 이 기록을 기기를 바꿔도 잃지 않도록 저장/불러오기 할 수 있는

정적 웹 트래커를 제공한다.

### 1.3 비목표(Out of Scope)
- 실제 티켓 예매/결제 기능 (예매는 티켓링크 등 외부 링크로 연결만 한다)
- 회원가입/로그인 기반 서버 계정 시스템 (레퍼런스와 동일하게 클라이언트 로컬 저장 우선)
- 캐스팅 자동 실시간 크롤링 (제작사 공지가 자주 바뀌므로 수동 업데이트 전제, 단 구조는 갱신하기 쉽게 설계)

### 3.5 다회 관람 혜택 참고 사례 — "내 재관판"(유사 서비스 스크린샷)
사용자가 제공한 스크린샷은 별도 서비스(도장판 관리 앱으로 추정)의 "내 재관판" 화면으로, 아래와 같은 구조를 갖는다.
- 공연 카드: 공연명, 기간, 진행률(🎫 5/7), 진행 바(progress bar)
- **적립** 섹션: 실제 관람할 때마다 도장이 하나씩 찍히고, 그 아래 날짜(예: 06.18, 06.20, 06.28, 07.02, 07.02)가 기록됨. 미달성분은 빈 원(6, 7)으로 표시되고 혜택 달성 지점(🎁)에는 선물 아이콘 배지가 붙는다.
- **혜택** 섹션: 임계 횟수(예: 5회)에 도달하면 "수령 가능" 상태로 바뀌며 수령하기 버튼 활성화, 미달성 혜택(7회)은 🔒 잠금 + "n회 달성 시" 문구로 표시.
- 유의사항: "재관판은 실제 혜택 적립/수령 처리가 아니며, 적립 기록과 혜택 진행 상황을 **직접 관리**하기 위한 기능"이라는 명시적 문구 — 즉 이 기능은 공식 MD 카드의 물리적 도장 적립을 사용자가 스스로 복기/시뮬레이션하는 개인 기록용 도구이지, 실제 수령을 대행하는 기능이 아니다. **우리 트래커의 "출동 기록 카드" 탭도 동일한 성격(개인 기록용 미러링)으로 설계한다.**

## 4. 사용자 & 사용 시나리오

### 4.1 타겟 사용자
- 〈사칠〉을 다회차 관람(회전문)하는 팬
- 특정 배우(안정원 역 또는 강이준 역 특정 배우)의 회차를 우선적으로 챙겨보고 싶은 사용자
- 관람 기록을 개인 아카이빙 목적으로 남기고 싶은 사용자

### 4.2 핵심 시나리오 (탭 흐름 기준)
1. **[스케줄 탭] 회차 탐색**: 사용자가 사이트 접속 → 상단에서 전체 배우/배역 정보를 확인하고, 캐스팅 스케줄 표에서 날짜/시간/캐스팅을 확인한다.
2. **[스케줄 탭] 배우 필터링**: 관심 배우(예: 김재한)를 탭에서 선택 → 해당 배우가 출연하는 회차만 하이라이트된다. 📌로 고정하면 다음 방문 시에도 유지된다.
3. **[스케줄 탭] 관람 기록**: 실제로 본(또는 예매한) 회차 행 오른쪽의 좌석 입력 필드에 좌석 번호를 입력한다 → 자동으로 해당 회차가 "관람 회차"로 등록되고 회차 정보(날짜, 캐스팅)가 함께 저장된다.
4. **[정산 탭] 좌석 이력 확인**: 좌석배치도 SVG/Canvas 위에 지금까지 입력한 모든 좌석이 마커로 표시된 것을 확인하고, 관람 회차 리스트(날짜·캐스팅)를 스크롤로 훑어본다.
5. **[출동 기록 카드 탭] 혜택 진행률 확인**: 지금까지 등록된 관람 회차 수를 기준으로 5회/7회 혜택까지 얼마나 남았는지 도장판 형태로 확인한다.
6. **백업/이전**: 브라우저를 바꾸거나 초기화 전, 데이터 저장(export) → 이후 데이터 불러오기(import)로 복원한다.

## 5. 정보 구조(IA) & 기능 요구사항

### 5.0 전체 IA — 모바일 최적화 하단/상단 탭 3종

본 서비스는 **모바일 우선(mobile-first)**으로 설계하며, 화면 최상단 또는 하단에 고정되는 3개 탭으로 전체 기능을 구분한다. 데스크톱에서는 동일 탭을 상단 가로 탭바로 표시한다.

```
┌─────────────────────────────┐
│   사칠 관람 트래커 (헤더)     │
├─────────────────────────────┤
│                               │
│      (탭별 컨텐츠 영역)       │
│                               │
├─────────────────────────────┤
│  [스케줄] [정산] [출동기록카드] │  ← 탭 네비게이션 (모바일: 하단 고정)
└─────────────────────────────┘
```

| 탭 | 핵심 역할 |
|---|---|
| **① 스케줄** | 전체 배우/캐스팅 정보 열람 + 회차별 좌석 입력(관람 기록의 입구) |
| **② 정산** | 스케줄 탭에서 입력한 데이터를 기반으로 좌석 이력(좌석도 마커)과 관람 회차 리스트를 조회 |
| **③ 출동 기록 카드** | 관람 회차 수를 기준으로 다회 관람 혜택(5회/7회) 달성 현황을 도장판 형태로 시각화 |

세 탭은 **동일한 단일 소스(관람 회차 로그)를 공유**하며, 스케줄 탭에서의 입력이 정산·출동 기록 카드 탭에 실시간 반영되는 구조다(아래 6장 데이터 모델 참고).

---

### 5.1 ① 스케줄 탭

| ID | 기능 | 상세 요구사항 |
|---|---|---|
| F-01 | 공연 정보 헤더 | 공연명, 기간, 장소, 예매처(티켓링크) 링크 표시 |
| F-02 | 전체 배우 정보 | CHARACTER & CAST 형태로 배역(안정원/강이준)별 태그라인 + 배우 3인 프로필(이미지, 이름) 노출 |
| F-03 | 배우 필터 탭 | 안정원 역 3인 + 강이준 역 3인, "전체 보기" 포함. 탭 선택 시 해당 배우 출연 회차 행 하이라이트 |
| F-04 | 배우 고정(핀) | 필터 선택 상태를 로컬에 저장해 재방문 시 유지 |
| F-05 | 캐스팅 스케줄 테이블 | 월별/주별 그룹핑, 날짜·요일·시간·안정원 배우·강이준 배우 컬럼. 공연 없는 날은 "공연 없음"으로 표시 |
| F-06 | **좌석 입력 필드 (회차 행 오른쪽)** | 각 캐스팅 스케줄 행 오른쪽 끝에 좌석 입력용 텍스트 필드 배치(예: "1층 F열 12"). **값이 입력되면 해당 회차가 자동으로 "관람함" 상태로 전환**되고 관람 회차 로그에 날짜·시간·캐스팅 정보가 함께 기록된다 |
| F-07 | 관람 여부 토글 | 좌석 미입력이어도 수동으로 "관람함" 체크만 가능(좌석 정보를 아직 모를 때 대비) |
| F-08 | 첫공/막공 배지 | 데이터에 플래그가 있는 회차에 배지 표시 |
| F-09 | 면책 문구 | "상기 스케줄은 배우 및 제작사 사정에 따라 사전 공지 없이 변경될 수 있습니다." 하단 고정 |

### 5.2 ② 정산 탭

| ID | 기능 | 상세 요구사항 |
|---|---|---|
| F-10 | **좌석 정보 — SVG/Canvas 좌석 마커** | 링크아트센터 드림 2관 좌석배치도를 SVG(또는 Canvas)로 렌더링하고, 스케줄 탭에서 입력한 좌석 좌표마다 마커(예: 📍 또는 색상 점)를 표시. 여러 회차 관람 시 마커가 누적되어 표시되며, 마커 클릭/탭 시 해당 좌석의 관람 날짜·캐스팅 정보를 툴팁으로 노출 |
| F-11 | 좌석 텍스트 폴백 | 좌석 좌표를 자동 매핑할 수 없는 경우(좌석표 데이터 미비 시), 좌석 리스트를 텍스트 목록으로도 함께 제공 |
| F-12 | **관람 회차 리스트** | 사용자가 관람 처리한 모든 회차를 날짜순으로 리스트업. 각 항목에 날짜·시간·안정원 배우·강이준 배우·좌석 정보 표시 |
| F-13 | 배우별 집계(부가) | 관람 회차 리스트를 배우 기준으로 재집계하여 "n / 전체 출연 회차" 표시 (레퍼런스 사이트의 기존 기능 유지) |
| F-14 | 데이터 저장/불러오기 | 사용자의 좌석·관람 회차·핀 상태를 파일(JSON) 또는 브라우저 저장소로 export/import |

### 5.3 ③ 출동 기록 카드 탭

〈사칠〉 공식 MD "출동 기록 카드"(§3.6) 컨셉을 그대로 미러링하는 **개인 기록·시각화 전용** 탭이다(§3.5 "재관판" 사례와 동일하게, 실제 혜택 수령을 대행하지 않음을 UI 문구로 명시).

| ID | 기능 | 상세 요구사항 |
|---|---|---|
| F-15 | 진행률 요약 카드 | 상단에 공연명, 기간, 총 관람 회차 수(🎫 n/7 형태), 진행 바(progress bar) 표시 |
| F-16 | 도장판(적립) 영역 | 관람 회차 로그의 건수만큼 도장 아이콘이 채워지고, 각 도장 아래 관람 날짜 표시. 미달성 칸은 빈 원으로, 혜택 임계치(5, 7회)에는 🎁 배지 표시 |
| F-17 | 혜택 리스트 & 달성 여부 | "5회 관람 시 — 지정 폴라로이드 1매", "7회 관람 시 — 실황 OST" 두 항목을 표시하고, 관람 회차 수 ≥ 임계치이면 "달성" 상태(강조 스타일)로, 미달성이면 🔒 + "n회 달성 시" 문구로 표시 (실제 "수령하기" 버튼은 제공하지 않고 "달성함 ✅" 배지만 표시 — 실물 카드 적립을 대체하지 않기 때문) |
| F-18 | 안내 문구 | "본 기록은 개인 관람 이력을 정리하기 위한 참고용이며, 실제 MD 부스 적립·혜택 수령을 대신하지 않습니다." 고정 노출 |

## 6. 데이터 모델 (v0.2)

정적 공연 데이터(`show`)와 사용자 로컬 상태(`userState`)를 분리하는 원칙은 유지하되, **좌석 마커(정산 탭)와 출동 기록 카드(3번째 탭)가 모두 동일한 `viewingLog`를 파생 데이터로 사용**하도록 설계한다.

### 6.1 정적 공연 데이터
```json
{
  "show": {
    "title": "사칠",
    "period": { "start": "2026-08-04", "end": "2026-10-25" },
    "venue": "링크아트센터 드림 2관",
    "bookingLinks": [
      { "name": "티켓링크", "url": "https://www.ticketlink.co.kr/product/64013" }
    ],
    "benefitTiers": [
      { "count": 5, "reward": "지정 폴라로이드 1매", "note": "원하는 배우 지정 후 수령 가능" },
      { "count": 7, "reward": "실황 OST", "note": "트랙리스트 추후 공지" }
    ]
  },
  "roles": [
    { "roleId": "jungwon", "roleName": "안정원", "tagline": "나는 무엇을 잊고 있는가", "actors": ["황민수", "선한국", "김찬종"] },
    { "roleId": "yijun", "roleName": "강이준", "tagline": "누구 잘못도 아니야", "actors": ["임태현", "박정혁", "김재한"] }
  ],
  "performances": [
    {
      "id": "2026-08-04-2000",
      "date": "2026-08-04",
      "weekday": "화",
      "time": "20:00",
      "cast": { "jungwon": "황민수", "yijun": "임태현" },
      "tags": []
    }
  ],
  "seatMap": {
    "svgUrl": "/assets/link-artcenter-dream2-seatmap.svg",
    "seatCoordinates": {
      "1층 F열 12": { "x": 214, "y": 118 }
    }
  }
}
```

### 6.2 사용자 로컬 상태 (`userState`, 정적 데이터와 분리)
```json
{
  "pinnedActors": ["김재한"],
  "viewingLog": [
    {
      "performanceId": "2026-08-04-2000",
      "date": "2026-08-04",
      "cast": { "jungwon": "황민수", "yijun": "임태현" },
      "seat": "1층 F열 12",
      "checkedAt": "2026-08-04T20:05:00+09:00"
    }
  ]
}
```

**파생 데이터 (derived, 저장하지 않고 매번 계산):**
- 정산 탭 좌석 마커 = `viewingLog[].seat` → `seatMap.seatCoordinates`에서 좌표 조회 → SVG/Canvas에 마커 렌더
- 정산 탭 관람 회차 리스트 = `viewingLog`를 `date` 기준 정렬
- 배우별 집계 = `viewingLog`를 `cast.jungwon` / `cast.yijun` 기준으로 그룹핑한 개수
- 출동 기록 카드 진행률 = `viewingLog.length` vs `show.benefitTiers[].count`

> ⚠️ `performances` 전수 데이터(§3.4)와 `seatMap.seatCoordinates`(좌석배치도 좌표)는 아직 미확보 상태이며, 개발 착수 전 반드시 채워야 하는 핵심 블로커 데이터다.

## 7. 비기능 요구사항

- **모바일 우선(Mobile-first)**: 1차 타겟은 모바일 브라우저이며, 탭 네비게이션은 엄지로 조작하기 쉬운 하단 고정을 기본으로 한다. 데스크톱은 반응형으로 상단 탭바 전환.
- **정적 호스팅**: GitHub Pages 등 정적 사이트로 배포 가능해야 함 (레퍼런스와 동일)
- **반응형 테이블**: 스케줄 탭의 캐스팅 표는 좁은 화면에서 좌석 입력 필드까지 포함해도 가로 스크롤로 정상 열람 가능해야 함
- **SVG/Canvas 성능**: 정산 탭 좌석 마커 렌더링은 좌석 수가 많아도(수백 석) 버벅임 없이 동작해야 하며, 모바일 터치 확대/축소를 지원한다
- **데이터 영속성**: 로컬 저장소(localStorage) 기반이며, 브라우저 캐시 삭제 시 유실 위험 → export/import 기능 필수
- **탭 간 데이터 일관성**: 스케줄 탭에서의 좌석 입력/체크가 정산·출동 기록 카드 탭에 즉시(같은 세션 내) 반영되어야 함
- **접근성**: 텍스트 대비, 터치 타겟 크기 등 기본 접근성 고려
- **유지보수성**: 공연 회차 데이터·좌석표 좌표는 코드와 분리된 JSON/데이터 파일로 관리해 향후 캐스팅 변경 시 쉽게 갱신 가능해야 함
