# 뮤지컬 〈사칠〉 관람 트래커 — CLAUDE.md

## 개요
PRD(`사칠_관람트래커_PRD.md`)와 디자인 핸드오프(zip 내 `.dc.html` 프로토타입)를 기반으로 구현한 **모바일 우선 정적 웹 트래커**. 빌드 도구·프레임워크 없이 순수 HTML/CSS/Vanilla JS로 작성되어 GitHub Pages 등 정적 호스팅에 바로 배포 가능.


## 아키텍처 (js/app.js)
- **정적 데이터** `SHOW` (파일 상단): 공연 정보, 배역/캐스팅, 회차(`performances`), 혜택 티어(5회/7회). **사용자 상태와 분리** — PRD의 핵심 원칙.
- **사용자 상태** `state`: `localStorage['sachil-tracker-v2']`에 자동 영속화(`persist()`).
- **렌더**: `render()`가 탭별 HTML 문자열을 생성해 `#app`에 주입(가상 DOM 없음, full re-render).
- **이벤트**: `document`에 click/input/change 위임 + `data-action` 속성 매칭.

## 3개 탭 기능
| 탭 | 데이터 소스 | 주요 기능 |
|---|---|---|
| 스케줄 | `watched`, `seats` | 배우 필터/핀, 관람 체크, 좌석 입력·복사 |
| 정산 | `seats`(좌석맵) + `boards`(배우 정산) | A~L열 좌석맵, 배우별 진행바, JSON export/import/reset |
| 출동기록 | `boards` | 다중 도장판, 적립 모달, 5·7회 혜택 수령 |

> ⚠️ **설계상 중요**: 정산 탭의 "배우별 정산" 수치는 스케줄 탭의 관람 체크가 아니라 **출동기록 탭 도장판(`boards`)의 배우 입력값**을 합산한 것. 좌석맵만 스케줄 탭 `seats`를 사용. (디자인 핸드오프 명세 그대로 — 두 데이터 소스는 의도적으로 분리됨)

## 남은 작업 (PRD §8 미확보 데이터)
1. **전체 캐스팅 스케줄** — 현재 1주차 7회차 샘플만 반영. `SHOW.performances` 배열만 갱신하면 됨(다른 코드 수정 불필요).
2. **좌석배치도 실측 좌표** — 현재 좌석맵은 A~L열 근사 그리드(`buildSeatRows`). 링크아트센터 드림 2관 실제 배치도 확보 시 교체.
3. 첫공/막공 플래그, 프로모션 배지, 다크모드 토글 등 Nice-to-have(F-19~23) 미구현.

## 데이터 갱신 가이드
회차 추가 예시 (`js/app.js`의 `SHOW.performances`):
```js
{ date:'08.10', dow:'월', time:'20:00', jungwon:'선한국', yijun:'박정혁' }
```
- `badge:'막공'` 추가 시 배지 자동 표시.
- 배우 이름은 `roles[].cast`와 **정확히 일치**해야 필터·정산이 정상 집계됨.
- 회차 ID는 `date + '_' + time` 조합(`perfId()`)으로 생성되므로 같은 날 여러 회차는 시간으로 구분됨.
